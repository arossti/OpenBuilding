### HOW TO READ THIS TABLE

**Row identity (first 3 columns):** Project Type → Size Band → Compliance / Program Context
**Phase columns (left → right):** Each cell encodes four fields separated by `|`:
```
STATUS | ACTORS (across all tracks) | TOOL / STANDARD | DELIVERABLE / OUTPUT
```

**Three parallel tracks:**
| Track | Purpose | Visual treatment |
|---|---|---|
| **Statutory** | Roles and activities required by building code (NBC/OBC/BCBC) regardless of EC or voluntary program activity. Always present for the building type/size. | Muted / baseline |
| **Voluntary Programs** | Optional certification and incentive programs layered on the statutory baseline (Passive House, LEED, CaGBC ZCB, CHBA Net Zero, ENERGY STAR, CMHC, etc.). Many voluntary standards exist beyond those listed here (WELL, Living Building Challenge, Fitwel, etc.). | Accent / conditional |
| **Embodied Carbon** | EC/wbLCA assessment activity — sometimes statutory (Vancouver, Federal), sometimes voluntary, but always runs parallel to the upper two tracks. This is the primary focus of this app. | Primary / highlighted |

> *AR and PE are top-level roles that span all three tracks — statutory authority on Part 3, eligible as QP for EC work on all building types, and may hold voluntary credentials (LEED AP, CPHD, etc.). An SC may also be a QP and may hold any number of voluntary credentials.*

**Status codes (EC track status):**
| Symbol | Meaning |
|---|---|
| ❌ | No EC activity at this phase (Statutory track roles still active) |
| 🟡 | Voluntary — self-initiated, no program requirement |
| 🔵 | Conditional — required only if pursuing a specific program, incentive, or upper tier |
| 🟠 | Emerging — piloted or under active policy development; not yet enforceable |
| 🔴 | Mandatory — legally required by adopted code, bylaw, or program condition |

**Phase mapping (RAIC Handbook of Practice / OAA Document 600):**
| Phase | Label | Industry Phases |
|---|---|---|
| p1 | Pre-Design / Concept | PD (Pre-Design / Programming / Feasibility) |
| p2 | Design (SD/DD/CD) + Permit Application | SD (Schematic Design), DD (Design Development), CD (Construction Documents) |

**Jurisdictions:**
- **NBC** = National Model Code baseline (applies where PT has adopted NBC 2025)
- **PT-BC** = British Columbia (BC Building Code + Energy Step Code + CleanBC)
- **PT-ON** = Ontario (OBC + SB-12 + O. Reg. 332/12)
- **PT-QC** = Québec (CCQ — distinct adoption path)
- **PT-OTHER** = AB / SK / MB / Atlantic provinces (NBC-aligned, no EC requirements)
- **FN** = First Nations — some adopt NBC via band council resolution; not legislated by province but by local bands and councils. Building code adoption and enforcement varies significantly by community. Federal funding (ISC/CIRNAC) may trigger federal EC requirements.
- **MUNI-VAN** = City of Vancouver — *mandatory EC: Part 3 via VBBL s.10.4 + COV Addendum v1.0 (April 2025); Part 9 low-rise residential via ZEB Bulletin Appendix II (Jan 2026). BEAM/MCE2 mandated for Part 9.*
- **MUNI-TOR** = City of Toronto (Toronto Green Standard v4) — *EC at Tier 3–4*
- **MUNI-ON-OTHER** = Ontario municipalities with GDS: Markham, Mississauga, Ottawa, Richmond Hill, Vaughan — *GDS frameworks without numeric EC targets*
- **VOL** = Voluntary program (CHBA, LEED, Passive House, ENERGY STAR, CaGBC ZCB, etc.)
- **FIN** = Financing / incentive program (CMHC MLI Select, utility DSM, etc.)
- **FED** = Federal government buildings (Treasury Board Standard on Embodied Carbon — mandatory EC)
- **NO-PERMIT** = No building permit involved — modelling for research, feasibility, or non-regulatory purposes

> **Municipality scope note:** Only Vancouver and Toronto have substantive, enforceable EC requirements as of 2025-26. A small cluster of Ontario municipalities have GDS frameworks without numeric EC targets. All other Canadian municipalities (~690+) have no EC-related green standard.
>
> **First Nations note:** Building code adoption on First Nations reserves is not governed by provincial legislation. Some communities adopt NBC via band council resolution; others have no formal building code. Federal funding programs (ISC, CIRNAC, CMHC on-reserve) may carry EC requirements via the Treasury Board Standard. This is an area of active development.

## APPENDIX C — Professional Requirements for Design and General Review (Ontario Reference Model)

> **Primary source:** PEO/OAA Joint Bulletin *"Design and General Review Requirements for Buildings in the Province of Ontario"* (2019 revision). Architects Act R.S.O. 1990, c. A.26 | Professional Engineers Act R.S.O. 1990, c. P.28 | Ontario Building Code Act, 1992.
>
> **National applicability:** All Canadian PTs have equivalent legislation under their respective Architects Acts and Engineering Acts. Ontario is used as the reference model because the PEO/OAA Joint Bulletin is the most explicit published table. PT-specific notes are provided below.

### C.1 — When Are AR and PE Required? (Joint Bulletin Table Summary)

| Building Type / Size | Architect Required? | Engineer Required? | General Review Required? | Licences + E&O Required? |
|---|---|---|---|---|
| **Part 9** — ≤ 3 storeys + ≤ 600 m² (houses, small buildings) | ❌ **Exempt** — *Architects Act* s.11(3)(a) | ❌ Generally no — specific engineered elements (atypical loads, engineered wood products, HVAC systems) still require a P.Eng for those elements | ❌ Not required by OBC for Part 9 — recommended practice only | Only if AR/PE voluntarily engaged — then full professional obligations apply to their scope |
| **Part 3** — Multi-unit residential (> 3 storeys or > 600 m²) | ✅ **Required** — must be retained from project inception | ✅ **Required** | ✅ **Required** (OBC Div.C s.1.3.3) | ✅ OAA licence + PEO licence + Certificate of Authorization + mandatory E&O insurance |
| **Part 3** — Assembly, institutional, commercial | ✅ **Required** | ✅ **Required** | ✅ **Required** | ✅ As above |
| **Part 3** — Industrial (most types) | ✅ **Required** | ✅ **Required** | ✅ **Required** | ✅ As above |
| **Part 3** — Farm buildings, certain storage/utility structures | ❌ Exempt (*Architects Act* s.11(3)(c)–(f)) | ✅ **Required** | ✅ **Required** | ✅ PEO licence + CoA + E&O insurance |

### C.2 — Professional Obligations When AR or PE is Engaged

Regardless of whether engagement is mandatory or voluntary, once a licensed professional is involved, the following obligations are non-negotiable:

**Architects (OAA):**
- Valid OAA licence (individual) or Certificate of Practice (firm) in force
- Mandatory professional liability (E&O) insurance — minimum limits set by OAA
- Design prepared by or under direct supervision of the licensed Architect (not "rubber-stamped" from unlicensed work — this is professional misconduct and may constitute fraud)
- Seal affixed only to work the Architect is responsible for
- General Review commitment via Schedule A (design), B (constructor review), C (completion)
- OAA member must be retained *from project inception* — not added at permit stage

**Engineers (PEO):**
- Valid PEO licence (individual) — P.Eng designation
- Certificate of Authorization (CoA) if providing services through a firm
- Mandatory professional liability (E&O) insurance — PEO minimum: $500,000 per occurrence / $1,000,000 aggregate (verify current PEO requirements)
- Design prepared by or under direct supervision of the licensed P.Eng
- Seal affixed only to work within the engineer's competence
- General Review via OBC Schedules equivalent to AR schedules for engineering components
- Only a PEO licensee may provide services within the practice of professional engineering — this cannot be delegated to an unlicensed person

**Both professions:**
- Scope of practice disputes between AR and PE are resolved by the Joint Practice Board under *Architects Act* s.52(7) or *Professional Engineers Act* s.47(4) — not by the Building Official
- Overlapping or ambiguous scope should be resolved between the professions before permit submission
- Building Officials may forward non-conforming applications to OAA or PEO for review

### C.3 — Implications for EC Modelling Under Professional Engagement

| Scenario | Professional Requirement Implication |
|---|---|
| EC model prepared by unregulated SC/LCA Practitioner for a **voluntary program** (LEED, TGS, feasibility) | No professional obligation triggered by EC work alone. SC operates without mandatory licensure or E&O. |
| EC model submitted as part of a **development permit** application under TGS Tier 3–4 | Submitted to City Planning, not Building Official. No AR/PE stamp required on EC report specifically. However, if the report is coordinated with the permit drawings, it falls under the AR/PE's general oversight of the project. |
| EC model submitted as part of a **building permit** application (hypothetical future mandatory requirement) | Would likely require submission under AR or PE seal, triggering full professional obligations. This is the regulatory design challenge for future code-mandatory EC requirements. |
| AR or PE **voluntarily engaged** on a Part 9 project | Full professional obligations apply to their scope even though the project is exempt. They cannot claim the exemption to reduce their standard of care once engaged. |
| EC model prepared by EA | EA certification does not constitute a regulated professional designation. EA has no authority to seal documents. EC work by an EA is not a professional service in the regulated sense. An EA acting as an LCA practitioner on a Part 3 project would need to work under AR or PE supervision for any permit submission. |
