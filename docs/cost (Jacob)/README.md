# Cost

Research into incorporating cost data into BEAM, enabling builders to co-evaluate cost and carbon metrics when making material choices. This workstream leverages Andy Thomson's domain knowledge to inform how costing could work within the application.

## Contents

### `background/`

- **`Costing general feasibility study .md`** — Research tasks outlining the scope of investigation, including:
  - Pricing frameworks: RS Means (standardized estimating data) vs. MSRPs
  - Retail data acquisition methods (e.g., API integrations with major retailers)
  - Data quality and regional variability challenges
  - Update protocols for maintaining pricing data over time
  - Tool integration and display considerations
  - General feasibility of cost comparisons and payback period analysis

## Relationship to BEAM

Builders consistently need to weigh carbon impact against project cost. This workstream explores how BEAM can present cost-carbon tradeoffs for material and assembly choices, helping users make informed decisions without requiring a separate estimating workflow. Cost data is expected to be fetched from external sources at runtime; the specific data sources and integration approach are to be determined through this research.
